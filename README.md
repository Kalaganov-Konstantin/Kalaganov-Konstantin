## 🚀 Обо мне

**Backend-разработчик** с ~3 годами опыта в продакшене. Основной стек — **Python** (FastAPI, async SQLAlchemy, Redis), есть боевой опыт на **Go**.

Проектирую и запускаю с нуля микросервисы для **финтеха и IoT**: распределённые платежи с идемпотентностью и отказоустойчивостью, удалённое управление парком устройств, переход с монолита на событийную архитектуру через Kafka. Делаю упор на надёжность данных — Transactional Outbox, circuit breaker, дедупликация — и переиспользуемые внутренние библиотеки.

## 💼 Опыт

### 🏢 АТОЛ — Backend-разработчик
`дек 2025 — наст. время` · *Облачная микросервисная платформа управления парком фискальных устройств*

- Спроектировал и запустил с нуля как основной разработчик микросервис удалённого доступа к устройствам по **VNC**: жизненный цикл соединений, старт/стоп сессий по MQTT, автоочистка зависших сессий по таймауту, автовосстановление при обрыве, версионирование протокола; состояние сессий — в Redis.
- Разработал переиспользуемую внутреннюю библиотеку **MQTT-клиента** с очередью публикации, идемпотентной инициализацией и структурным логированием; настроил CI с автопубликацией в PyPI — библиотеку подключили **5+ сервисов**.
- Внедрил паттерн **Transactional Outbox** для гарантированной доставки событий, вынес outbox и relay в общую библиотеку.
- Реализовал массовые операции над устройствами: групповые обновления и импорт конфигурации с агрегированным статусом, фильтрацией и сортировкой.

<sub>**Стек:** Python 3.11+, FastAPI, SQLAlchemy 2.0 (async), Alembic, Pydantic v2, MQTT (aiomqtt / EMQX), Redis, Docker, GitLab CI/CD, Prometheus, Sentry</sub>

### 💳 B2B fintech-платформа — Backend-разработчик
`дек 2025 — май 2026` · *Мультитенантная платформа: платежи, бонусы, кэшбэк, отчёты, seamless-кошелёк*

- Спроектировал **каскадный оркестратор платежей** с маршрутизацией между провайдерами: при отказе запрос автоматически уходит на следующего.
- Реализовал **идемпотентность** платёжных операций: дедупликация по сигнатуре с временным окном, TTL на уровне записи с фоновой очисткой, сохранение состояния для безопасного повтора.
- Внедрил **circuit breaker** на базе Valkey для изоляции падающих провайдеров.
- Интегрировал **5 платёжных провайдеров**, включая seamless-кошелёк: идемпотентность через блокировки БД, верификация вебхуков, мультитенантность, расчёт дневных лимитов вывода с учётом таймзон.
- Реализовал систему кэшбэка с устранением гонок через `SELECT FOR UPDATE` и вторую версию API отчётов.

<sub>**Стек:** Python, FastAPI, Flask (legacy), SQLAlchemy 2.0, Alembic, Pydantic v2, Valkey/Redis, AWS (ECS, CloudFormation), boto3</sub>

### 🏠 Уфанет — Backend-разработчик
`мар 2024 — дек 2025` · *Экосистема «Умный домофон» из десятков микросервисов*

- Проектировал и вёл миграцию распределённого монолита на **событийную архитектуру**: перевод синхронных HTTP-вызовов на асинхронную обработку через Kafka, **500K+ событий/день** с батчингом и партиционированием.
- Разработал как один из основных контрибьюторов **сервис биллинга ЖКХ**: счета и тарифы с привязкой к домам/подъездам/квартирам, эквайринг, синхронизация через Kafka, напоминания и push на Celery, аналитика в ClickHouse.
- Поддерживал **сервис ключей доступа для 50K+ устройств**: активация и замена ключей, BLE, webhook на появление устройств, мониторинг состояния в реальном времени по MQTT.
- Реализовал **anti-corruption layer** между старым и новым домофонным сервисом с двусторонней синхронизацией договоров — миграция без простоя.
- Написал на **Go** сервисы домофонии поверх MQTT: подписки на устройства с восстановлением и конкурентной моделью на мьютексах, интеграция звонков и SIP-мониторов.
- Настроил CI/CD, мониторинг на Prometheus/Grafana, трейсинг на Jaeger. Провёл **50+ код-ревью**, менторил **3 джунов** до уровня миддл.

<sub>**Стек:** Python, Go, Django, DRF, FastAPI, SQLAlchemy 2.0 (async), Celery, Kafka, MQTT, gRPC, PostgreSQL, ClickHouse, Redis, Docker, Kubernetes, Prometheus, Grafana, Jaeger</sub>
